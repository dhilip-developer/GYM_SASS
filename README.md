# GymOS SASS Setup Guide

Welcome to the GymOS repository! 

## ⚠️ Important Note on Security & Credentials

For security reasons, several files have been intentionally excluded from this repository and must be created manually on your local server.

### 1. Google OAuth Credentials
The `server/credentials.json` file is **NOT** included in this repository. GitHub's Secret Scanning automatically blocks this file to protect your Google Cloud account from being compromised. 
**To set up the server:**
- Create a `credentials.json` file inside the `server/` directory.
- Paste your Google OAuth 2.0 Client ID and Client Secret into this file.

### 2. WhatsApp Agent Installer (`.exe`)
The compiled `GymOS Agent Setup 1.0.0.exe` is **NOT** included in this repository. 

**Why isn't it here?**
The installer packages a full Node.js runtime, the Electron engine, and heavy WhatsApp cryptographic libraries, resulting in a file size of **~102 MB**. GitHub has a strict **100 MB file limit** for source code, so it cannot be pushed here.

**How to build and distribute the agent `.exe`:**
Any time you need to generate a fresh installer for a client, run these commands in your terminal:
```bash
cd agent
npm install
npm run build
```
Once the compilation finishes (it takes a few minutes), you will find the `GymOS Agent Setup 1.0.0.exe` inside the `agent/dist/` folder. 
You can then share that `.exe` with your gym clients via Google Drive or by uploading it manually to the **GitHub Releases** tab on this repository.

## Architecture Overview

GymOS operates on a hybrid architecture to maximize both flexibility and security:

1. **Cloud Backend (`server/`)**: Manages the database, business logic, billing, and scheduling.
2. **Web Dashboard (`client/`)**: The React frontend where Gym Admins manage their software.
3. **Local WhatsApp Agent (`agent/`)**: A standalone, background `.exe` application that runs on the gym owner's physical computer. It securely polls the Cloud Backend for automated messages and sends them through a local WhatsApp session. This ensures cryptographic session keys never touch the cloud server and are saved locally in the user's `C:\` drive (`AppData/Roaming/gymos-whatsapp-agent`).

## Comprehensive Features & Pages Guide

GymOS is a multi-tenant SaaS platform. There are two primary roles: the **Super Admin** (who manages the software platform) and the **Gym Admins** (who manage their individual gyms).

### 🏢 Super Admin Features
The Super Admin dashboard is the central command center for the software owner.
- **Super Admin Dashboard:** Get a bird's-eye view of total revenue, active gym subscriptions, and platform growth.
- **Gyms Management:** View a list of all registered gyms, manage their subscription tiers (Basic, Pro, Elite), and suspend inactive accounts.
- **Revenue Report:** Detailed analytics of SaaS subscription revenue generated by the gyms using the software.

### 🏋️ Gym Admin Features
When a Gym Owner logs into their specific gym portal, they have access to a full suite of management tools:

- **Dashboard:** A quick summary of total active members, revenue for the month, recent sign-ups, and upcoming expiries.
- **Members:** Full CRM to add, edit, and view member profiles. Includes health details, emergency contacts, and attendance history.
- **Memberships & Billing:** Create custom membership plans (e.g., 1 Month, 3 Months, Annual). Track payments, view invoices, and manage active subscriptions for members.
- **WhatsApp Campaigns:** A powerful marketing tool to send bulk promotional messages to active, inactive, or all members.
- **WhatsApp Session (Agent Integration):** The secure setup page where gym owners link their local WhatsApp Agent `.exe` by scanning a QR code directly from their browser.
- **Trainers & Staff:** Manage employees, assign trainers to specific members, and track staff attendance.
- **Leads Management:** A Kanban-style board to track prospective members from initial inquiry to final conversion.
- **Announcements:** Post notices and updates that are instantly visible on the gym's public noticeboard or member app.
- **Settings:** Configure gym details, business hours, GST/tax settings, and WhatsApp automation preferences (e.g., enabling 8:00 AM automated expiry reminders).

### 🤖 Automated WhatsApp Workflows
GymOS features a fully automated background workflow to keep gyms connected with their members:
- **Daily Expiry Reminders:** Every day at 8:00 AM, the cloud server checks for members whose plans are expiring in 7 days, 3 days, or today.
- **Local Syncing:** Instead of sending from the cloud (which triggers spam filters), the server queues these messages.
- **Agent Execution:** The `GymOS Agent Setup.exe` running on the gym owner's local computer automatically pulls this queue and sends the personalized reminder messages securely from their own business WhatsApp number.
